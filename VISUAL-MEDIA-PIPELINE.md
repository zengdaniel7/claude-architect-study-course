# Visual Media Pipeline

## Rule

Every lesson communicates its main idea visually before sustained reading.

## Media choices

- **Real screenshots:** Finder, TextEdit, Terminal, GitHub, Claude Code, API consoles, and working outputs when interface recognition matters
- **Canva:** memory maps, architecture pictures, flashcard decks, and printable review sheets
- **Higgsfield images:** one-scene concept stills for ideas that cannot be captured from a real interface
- **Higgsfield motion:** short state-change sequences such as request -> tool call -> result -> next turn
- **HeyGen:** short captioned narration over a useful screen demo or teaching visual
- **Local HTML:** an immediate interactive fallback when external media is unavailable

## Minimum for a required unit

1. one useful visual or real screenshot
2. listen control or narrated clip
3. captions or a short transcript
4. one boxes-and-arrows drawing prompt
5. alt text that states the teaching point

No autoplay. No dense body copy inside an image. Every video needs a still or text fallback. Generated interfaces must never be presented as real product screenshots.

## Reviewed video rule

- Review long videos transcript-first, then inspect frames around demonstrations.
- Link the exact lesson segment and keep a separate full-episode link.
- Use one primary source and at most one optional community clip per lesson.
- Label community material clearly; current official documentation wins when behavior differs.
- Keep captions, a one-line focus, and a local notes fallback.
