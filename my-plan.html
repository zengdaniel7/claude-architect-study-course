Fast CCA-F foundations

After the Workbench, confirm the core Claude words and move into real architecture practice.

  • API — a way for two programs to talk (you send a request, you get a response).
  • JSON — a simple text format for structured data, like { "name": "Dan" }.
  • Schema — the expected shape of data: which fields exist and what type each is.
  • CLI / terminal — typing commands to your computer instead of clicking.
  • Git — saves versions of your files so you can track and undo changes.
  • Token — a chunk of text (~¾ of a word). Models read and bill by tokens.
  • Model — the AI "brain" (e.g., Claude) that turns input into output.
  • Prompt — the instructions/question you give the model.
  • Context window — how much text the model can hold in mind at once.
  • Tool — an action you let the model call (search, run code, look up an order).
  • Agent — a model that loops: think → use a tool → read the result → repeat → finish.
  • MCP — a standard "plug" that connects Claude to outside tools and data.
  • Structured output — making the model return clean JSON that matches a schema.
