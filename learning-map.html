🛰️ The big picture — how a Claude app actually works

One request, end to end

Everything in the exam hangs off this one flow. Learn to draw it from memory.

Read it left→right: you ask → your app adds the API key and prompt → the model thinks inside its context window → it may call tools or read resources, looping until done → it returns an answer, often as JSON. Master this and 80% of the exam is "which box, and why".

🧱 System-design foundations for AI architects

The "holistic" layer — classic system-design ideas that the Reliability domain quietly tests. Skim these; they make everything else click.

① Request → Response (an API)

You send, you get back. Two programs talking. The key is secret.

② JSON = data with labels

A labelled form. It's how data moves in and out of the API. One missing comma breaks it.

③ Real-time vs Batch

Pick batch when results can wait — it's cheaper. Real-time for anything blocking a person.

④ Caching + Retries (don't redo work / don't crash)

Two reliability staples: cache what doesn't change, retry transient failures instead of crashing.

🤖 Priority 1 — Agentic Architecture 27%

The Agent Loop — the single most important diagram

An agent = model + tools + a loop. It keeps going until stop_reason says it's done.

⚠️ Exam trap: the loop's real stop signal is stop_reason (tool_use → keep going, end_turn → finish) — not a fixed "max 5 turns" counter. A hard cap is a safety net, never the primary brain.

One agent vs. Orchestrator–Workers

When a job is big, a coordinator splits it and hands pieces to focused subagents.

⚠️ Two classic traps: (1) subagents don't automatically know what the coordinator knows — context must be passed explicitly. (2) Don't reach for multi-agent when one agent in a loop would do — coordination has a cost.

🧰 Frameworks vs raw API — the glass box and the magic box

Frameworks (Claude Agent SDK, LangChain, etc.) write the boring plumbing for you — but they hide what's really happening. Anthropic's advice: start with the glass box.

💡 Remember it as: glass box first, magic box later — and never trust a magic box you haven't opened. Exam angle: "start with direct API calls + simple composable patterns (chaining, routing, tools); add a framework only when you understand what it hides."

The Verification Ladder

How to trust an agent's work — most robust on top.

Provenance + Human-in-the-loop

Production systems must say where a fact came from, and route risky calls to a person.

⚠️ An answer with no source can't be verified — it just "sounds confident". Tag everything; escalate the uncertain.

✍️ Priority 3 — Prompting & Structured Output 20%

The Structured-Output pipeline

How to get validated JSON — and what to do when it isn't valid.

Nullable is the hero field: it lets the model say "not present" instead of making something up. Format error → retry with the error text. Real fact missing → human.
